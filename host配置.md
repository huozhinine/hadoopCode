# 如何在 Windows 上修改 hosts 文件 (用于Hadoop映射)

您需要修改 Windows 的 `hosts` 文件，来为您本地的 Java/IDE "欺骗" Windows，告诉它 `Master01` 和 `Master02` 这两个主机名对应的 IP 地址。

### 1. Hosts 文件的位置

hosts 文件位于以下路径：

C:\Windows\System32\drivers\etc\hosts

### 2. 您需要添加的内容

根据您之前的日志（显示了 `Master01/192.168.184.132` 和 `Master02/192.168.184.133`），您需要在 `hosts` 文件的**最底部**添加以下两行：

```
192.168.184.132  Master01
192.168.184.133  Master02
```

**提示**：您最好把**所有**集群节点（包括 `Slave01`, `Slave02`, `Slave03`）的 IP 和主机名映射都添加上，这可以防止以后 HDFS 客户端连接 DataNode 时出现类似的解析错误。

### 3. 详细修改步骤 (必须以管理员身份)

您**不能**直接双击打开该文件进行修改，Windows 会因为权限不足而阻止您保存。请**严格按照**以下步骤操作：

1. **以管理员身份打开“记事本”**

   - 点击 Windows 的“开始”菜单。
   - 输入 `记事本` (或 `Notepad`)。
   - 在搜索结果中的“记事本”上**右键单击**。
   - 选择 **“以管理员身份运行”**。

2. **打开 Hosts 文件**

   - 在管理员记事本中，点击左上角的 **“文件(F)” -> “打开(O)...”**。

   - 在顶部的“文件名”栏中，完整复制并粘贴以下路径，然后按回车：

     C:\Windows\System32\drivers\etc

   - 您可能会看到“未找到匹配项”。这是因为记事本默认只查找 `.txt` 文件。

3. **切换文件类型**

   - 在“打开”窗口的右下角，找到 "文本文档 (*.txt)"。
   - 点击它，并将其更改为 **"所有文件 (*****.*****)"**。

4. **编辑和保存**

   - 现在您会看到 `hosts` 文件了。双击打开它。
   - 在文件的**最底部**，添加您的 IP 映射（如上所示）。
   - 点击 **“文件(F)” -> “保存(S)”** (或按 `Ctrl+S`)。
   - 由于您是以管理员身份运行的，文件会成功保存。

5. **验证 (可选)**

   - 打开您的 Windows 终端 (CMD 或 PowerShell)。
   - 运行 `ping Master01`。
   - 您应该会看到它回复 `来自 192.168.184.132 的回复...`，这证明映射已生效。
